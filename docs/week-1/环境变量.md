# 环境变量

## Q: 环境变量中读取配置 什么叫环境变量中?这环境变量是哪来的？

这是一个非常棒的问题。要理解“环境变量”，我们可以把计算机系统想象成一个巨大的**办公室**。

### 1. 什么是环境变量 (Environment Variables)？

简单来说，**环境变量**是操作系统层面的“全局变量”。它们就像是贴在办公室墙上的“公共便签”，上面记录着系统运行所需的一些重要信息。

- **“环境”**：指的是你的程序运行时的背景环境（就像你在办公室、家里还是户外工作）。
- **“变量”**：指的是这些信息是可以随时更改的，比如 `PORT`（端口号）、`DB_URL`（数据库地址）等。

在你的 Go 代码中，`os.Getenv("PORT")` 的意思就是：**“去操作系统的墙上找一张写着 PORT 的便签，把上面的值拿给我。”**

---

### 2. 环境变量是从哪来的？

环境变量主要有三个来源：

#### A. 操作系统预设

当你打开电脑时，Windows、macOS 或 Linux 就已经自带了一些环境变量。

- 例如 `PATH`：告诉系统去哪里找可执行程序。
- 例如 `USER`：当前登录的用户名。

#### B. 用户手动设置

程序员可以根据需要自己创建。比如你的代码里需要 `PORT`，你就可以手动在系统中设置它：

- **在 Linux/macOS 终端：** `export PORT=9090`
- **在 Windows PowerShell：** `$env:PORT = "9090"`
- **在 Windows 界面：** 通过“系统属性 -> 环境变量”面板手动添加。

#### C. 容器或云平台（最常用的场景）

现代软件通常跑在 Docker 或云服务器（如 AWS、阿里云）上。

- 在启动容器时，我们会传入环境变量：`docker run -e PORT=8080 my-go-app`。
- **为什么要这么做？** 这样你就不需要为了改一个端口而修改代码、重新编译了。只需要在启动命令里改一下环境变量，代码通过 `os.Getenv` 就能自动读取到新值。

---

### 3. 为什么要用环境变量，而不是直接写在代码里？

这主要涉及到 **“代码与配置分离”** 的原则：

1. **安全性**：你绝对不想把数据库密码、API 密钥直接写在代码里并上传到 GitHub。通过环境变量，你可以把敏感信息留在服务器本地。
2. **灵活性**：你的代码在“开发环境”可能用 8080 端口，但在“生产环境”可能需要用 443 端口。如果你硬编码在代码里，每次换环境都要改代码。
3. **一致性**：同一个程序编译出来的二进制文件，可以通过给它不同的环境变量，让它扮演不同的角色。

### 总结

在你的代码逻辑里：

1. 程序运行。
2. 程序问操作系统：“墙上贴着 `PORT` 吗？”
3. **如果有**（比如你运行前设置了 `export PORT=9090`），程序就用 `9090`。
4. **如果没有**，程序就用你写的默认值 `:8080`（这就是 `fallback` 的意义）。
